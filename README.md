# Полная инструкция по установке и запуску проекта "Мультимодальные транспортные системы"

Этот проект представляет собой веб-приложение для расчета эффективности складских операций в транспортных системах. Оно состоит из двух частей: бэкенда (серверной части на Python) и фронтенда (веб-интерфейса на React). Следуйте этой инструкции шаг за шагом, даже если вы никогда не программировали.

## Шаг 1: Установка необходимого программного обеспечения

### 1.1 Установка Python

Python - это язык программирования, на котором написана серверная часть проекта.

1. Откройте браузер и перейдите на официальный сайт Python: https://www.python.org/downloads/
2. Скачайте версию Python 3.9.10
3. Запустите установщик
4. **ОБЯЗАТЕЛЬНО** отметьте галочку "Add Python to PATH" во время установки
5. Нажмите "Install Now" и дождитесь завершения установки
6. Проверьте установку: откройте командную строку (Win + R, введите `cmd`, Enter) и введите `python --version`

### 1.2 Установка Node.js

Node.js нужен для запуска веб-интерфейса.

1. Перейдите на https://nodejs.org/
2. Скачайте LTS версию (рекомендуется)
3. Запустите установщик и следуйте инструкциям
4. Проверьте установку: в командной строке введите `node --version` и `npm --version`

### 1.3 Установка Git (опционально, но рекомендуется)

Git нужен для скачивания проекта.

1. Перейдите на https://git-scm.com/downloads
2. Скачайте и установите Git для Windows
3. Проверьте: в командной строке введите `git --version`

## Шаг 2: Скачивание проекта

### Вариант 1: Используя Git (рекомендуется)
```
git clone https://github.com/AnneAlekseeva/multimodal-system.git
cd multimodal-system
```

### Вариант 2: Скачивание ZIP архива
1. Перейдите на страницу репозитория GitHub
2. Нажмите зеленую кнопку "Code" → "Download ZIP"
3. Распакуйте архив в удобную папку
4. Откройте командную строку в этой папке (Shift + правой кнопкой мыши → "Открыть окно PowerShell здесь")

## Шаг 3: Настройка и запуск бэкенда (серверной части)

### 3.1 Создание виртуального окружения

Виртуальное окружение изолирует зависимости проекта от остальной системы.

```
python -m venv venv
venv\Scripts\activate
```

### 3.2 Установка зависимостей

```
pip install -r requirements.txt
```

### 3.3 Запуск сервера

```
uvicorn backend.main:app --reload
```

Сервер запустится на http://127.0.0.1:8000
Не закрывайте это окно командной строки!

## Шаг 4: Настройка и запуск фронтенда (веб-интерфейса)

Откройте **новое** окно командной строки и перейдите в папку frontend:

```
cd frontend
npm install
npm run dev
```

Фронтенд запустится на http://localhost:5173
Не закрывайте это окно!

## Шаг 5: Использование приложения

1. Откройте браузер (Chrome, Firefox, Edge)
2. Перейдите на http://localhost:5173
3. Вы увидите веб-интерфейс приложения
4. Введите необходимые параметры для расчета
5. Нажмите кнопку запуска расчета
6. Результаты отобразятся на странице

## Шаг 6: Остановка приложения

Чтобы остановить приложение:
- В окнах командной строки нажмите Ctrl + C
- Закройте окна

## Возможные проблемы и решения

### Проблема: "python не является внутренней или внешней командой"
**Решение:** Python не добавлен в PATH. Переустановите Python, обязательно отметив "Add Python to PATH"

### Проблема: Ошибка при установке зависимостей
**Решение:** Убедитесь, что виртуальное окружение активировано (в командной строке должно быть `(venv)`)

### Проблема: Порт 8000 или 5173 занят
**Решение:** 
- Закройте другие приложения, использующие эти порты
- Или измените порт в командах запуска

### Проблема: Frontend не может подключиться к backend
**Решение:** Убедитесь, что оба сервера запущены и работают

### Проблема: Ошибки с кодировкой (если на русском языке)
**Решение:** В командной строке введите `chcp 65001` перед запуском

## Что делает это приложение?

Приложение моделирует работу складского комплекса:
- Рассчитывает оптимальные графики обработки транспортных средств
- Сравнивает регулируемые и нерегулируемые потоки
- Показывает экономическую эффективность
- Генерирует отчеты и графики

## Дополнительная информация

- **Документация API:** Когда бэкенд запущен, перейдите на http://127.0.0.1:8000/docs для интерактивной документации
- **Логи:** Все сообщения об ошибках появляются в окнах командной строки
- **Файлы проекта:** Результаты расчетов сохраняются в папку `backend/output/`
